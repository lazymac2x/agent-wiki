# 03 · 1인 개발 운영룰 (자율경계 · 페이아웃 · 도구매핑)

> 이 노트는 **운영 계약 템플릿**이다. 구체 경로/값은 `<당신의 ...>` placeholder — 자신의 SOT 경로/결정규칙으로 채워라.

당신의 에이전트가 당신 대신 일할 때의 **운영 계약**. 구체 값(자격증명·계정ID·페이아웃 잔액·수익 숫자)은 **여기 살지 않는다** — `<당신의 자격/수익 SOT 파일>`이 owns. 이 노트는 그 SOT의 **구조와 결정규칙만** 미러(복붙 아님 — [[memory-bridge]]). 값이 필요하면 항상 SOT를 열어 읽는다.

## 자율결정 경계 (무엇을 혼자 하나)
| 행위 | 판정 | 근거 |
|---|---|---|
| 읽기·조사·빌드·로컬테스트·문서 | **자율** | 가역·저위험 |
| 코드 수정·라운드 진행·에뮬 두눈 검증 | **자율(검증 동반)** | 검증-선-영속(가짜0 금지) |
| **prod 쓰기**(배포·DB write·릴리스) | **HITL** | 비가역. AskUserQuestion 명시인증 먼저 |
| **거래 주문**(자동매매 실주문) | **HITL** | 비가역·금전. 음의엣지 검증 선행 |
| 페이아웃·출금·결제수단 변경 | **HITL** | 금전·비가역 |
| confidence < 80~90% / 모호 | **HITL** | 어느 루프든 게이트 |

🔑 규칙: **prod-write · 비가역 · 금전 · 저신뢰 = AskUserQuestion 먼저.** 나머지는 자율 진행 후 마일스톤에서 [[notify]]로 1줄 보고.

## HITL 게이트 운영
- 인증은 **행위 직전** 명시적으로(포괄 위임 금지). 인증 후 배포 명령(예: `wrangler`)/MCP 직접 실행.
- prod DB 수정은 **임시행 삽입 → 캡처/검증 → 정확 롤백** 패턴(흔적 최소). 파괴적 머지는 이동→삭제→rekey 순서.
- 배포 후 마일스톤(Act 완료·배포·머지)마다 알림 채널(예: 텔레그램)로 1줄 → [[notify]].

## 페이아웃 · 결제채널 (포인터만 — 복붙 금지)
숫자·계정·매트릭스는 SOT가 owns. 구조만:
- `<당신의 자격/수익 SOT 파일>` — 자격증명·계정ID·페이아웃·현재수익·자율결정 경계.
- `<당신의 결제채널 매트릭스 파일>` — 결제 채널 매트릭스(예: Gumroad·Polar 등 중립 나열).
- 수익 목표/진척 추적 = 당신의 수익/목표 추적 스킬 **호출**(재구현 금지).
> ⚠️ 결제채널 수수료·가용성은 변한다. 채널 선택 전 항상 매트릭스 SOT를 다시 읽는다(stale=오도).

## 도구 매핑 (작업 → 도구)
| 작업 | 도구 | 비고 |
|---|---|---|
| 엣지/서버리스(Workers/DB/스토리지) | 엣지 플랫폼 CLI/MCP(예: Cloudflare `wrangler` / Vercel / Deno) | DB = ground truth(앱 통계 의심 시 직접 쿼리) |
| 자동매매 | 증권사 API(벤더중립) | 실주문 = HITL. 서킷브레이커 우회 금지 |
| 스크래핑·자동화 | 데이터 파이프라인/수집 도구 | SOT=`<당신의 파이프라인 인벤토리 파일>`(인벤토리·SLA·incident) |
| 디바이스 제어·GUI·배포 | 당신의 디바이스 제어 스킬 | 샌드박스 우회 |
| 전수조사→무한 90점 | 당신의 심층-피드백/적대비평 스킬 | harness 문서가 사용설명서 |
| 런타임 자율수복 | 당신의 런타임 자가수복 루틴 | ≈ [[memory-bridge]]가 가리키는 self-heal |
| 멀티에이전트 풀스택 | 당신의 멀티에이전트 빌드 스킬 | PM/Architect/Mobile/Backend/QA |
| 교차검증 다관점 토론 | 당신의 다관점 토론 스킬 | 여러 페르소나 교차검증 |
| UI/로고/색/디자인 | 디자인 전문 에이전트/스킬에 위임 | crude 자가처리 금지(벡터 우선) |

🔑 기존 skill은 **호출·재사용**(중복구현 금지). agent-wiki harness 문서는 그 브리지다.

## 빌드 · 배포 규율
- 빌드 = GUI 빌드큐 워처(kickstart 즉시발화). 데몬 타임아웃 회피 = `nohup &` 디태치.
- 이중트리(예: 앱+워커 분리): 앱 SOT=`<당신의 앱 소스 경로>`, 워커 SOT=[[api-endpoints]] 정본 경로, **단일 git DB 공유(경로한정 add)**. 라운드마다 양 트리 각 커밋 의무.
- `build-green ≠ live-works`: 결정론 테스트 + 에뮬 두눈으로 닫는다(무음스킵=vacuous green=금지).
- 자주 쓰는 명령 구문은 [[05-knowledge-library]]가 owns.
