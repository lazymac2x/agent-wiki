# 자동매매 / 자동화 봇 — 크로스프로젝트 공통

> 도메인 단일홈. 실거래 손실로 실증된 패턴만. 봇별 구현/상태는 [[example-bot]], 가드 일반론 [[guardrails]], 런타임 자율수복 [[self-healing]].

## 🔑 음의엣지를 먼저 검증한다 (수익은 구조적으로 가능한가)
- 점수/백테스트 이전에 **엣지가 양수인지** 본다. 음의엣지면 어떤 하네스도 수익 **구조적 불가**.
- 실증 예: 실현손익이 손익분기 미만(음의 엣지)·프로핏팩터 < 1·차익거래 수백 시간 **0체결**·오픈 포지션 큰 폭 마이너스 → "수익 90점" 구조적 불가 판정.
- "회수자본 0 = dust"는 **가치0 패배**(거래량/티켓 수가 아니라 실현 PnL이 진실).

## 🔑 서킷브레이커: 재시작 기록 경로 우회 금지
- 자해 재시작 루프 근본: **entry-stall**(체결 없음)을 "멈춘 루프"로 오인 + 재시작 큐가 **재시작 기록 함수를 미호출**해 브레이커를 우회 → 가드가 **매 재시작마다 소멸** → 큰 폭 재진입 손실.
- 수정 = 정상 무체결(AI 호출 0 = 킬스위치 OFF = 정상)을 멈춤으로 오판하지 않게 entry/alive 신호로 억제 + **모든 재시작 경로를 브레이커에 편입**(카운트가 영속돼야 다음 startup에서 차단).
- 가드는 **startup save로 materialize**돼야 한다. 메모리 영속만 하고 0청산이면 inert(무효). 가드 설계 [[guardrails]].

## 🔑 phantom 손실 = 부패 레코드 (은폐 아님) → sanitize
- 화면의 phantom 큰 손실은 실제 손실이 아니라 **부패 레코드**: 확률 범위 [0,1]을 벗어난 entry 값 등이 PnL 계산을 오염(실제 실현손익은 훨씬 작음).
- 모든 입력을 **sanitize**(확률 [0,1], 가격/수량 현실범위 게이트)한 뒤 집계. 오염값은 청산·배제(정직한 숫자가 진실).

## 🔑 단위테스트 green ≠ 라이브
- 결정론 유닛이 통과해도 **라이브는 다르다**: 가드가 메모리에만 살면(0청산) inert → startup save 시점에 비로소 materialize. 라이브 SUPPRESS를 실제로 관측해야 검증 완료.
- 적대 리뷰가 build-green·라이브정상이 통과시킨 결함(비용누수/계약위반)을 잡는다 → 코드회귀 축 별도 심사.

## prod 주문 = HITL
- 실주문/실송금은 **AskUserQuestion 명시인증 먼저**. 비가역. 주문은 거래소/브로커 API(주문 도구)로 라우팅하되 prod 체결 전 게이트. 현 상태/제약은 [[example-bot]].

## 안티패턴(하지 말 것)
- 음의엣지를 하네스 튜닝으로 덮기. · 재시작 경로가 브레이커 우회. · 부패값을 손실로 표시(또는 은폐). · 단위 green을 라이브로 단정. · dust 누적을 "포지션"으로 집계. 일반 안티패턴 [[anti-patterns]].
